# Agent Genome Lab — Terminology Glossary

This glossary normalizes the terminology used across the Agent Genome Lab, NVE, and the SkillGraph ecosystem. Using consistent language ensures clarity in both tooling and architectural discussions.

## Core Concepts

### 1. Incident
A **Failure Incident** (or simply **Incident**) is any deviation from expected behavior, an error, an hallucination, a timeout, or a sub-optimal outcome caused by an AI Agent during its operation. Incidents are the raw material for the learning pipeline. Once captured, an incident is documented in `MEMORY.md`.

### 2. EU (Evolutionary Unit)
An **Evolutionary Unit (EU)** is a crystallized piece of learning extracted from one or more incidents. It represents a single, verifiable lesson or strategy that the agent "learned" from its failure. An EU bridges the gap between raw failure data (Incident) and a reusable capability (Skill). EUs form the core of a Genome.

### 3. Genome
A **Genome** (or **Failure Genome**) is a structured markdown file (e.g., `001_failure_genome.md`) that documents the complete lifecycle of a specific failure or class of failures. It contains:
- The context and trace of the Incident(s).
- The root cause analysis (RCA).
- The extracted Evolutionary Units (EUs).
- The preventative patterns and anti-patterns derived from the failure.

### 4. Skill
A **Skill** is an actionable, executable capability or pattern that an agent can utilize to solve a task or avoid a failure. While Genomes are historical and analytical, Skills are operational. Skills are generated from EUs and are often represented in a structured `SKILL.md` format compatible with external ecosystems (like Anthropic's `skills.sh`).

### 5. Package (Module)
A **Package** is a self-contained bundle containing a combination of Genomes, EUs, and Skills around a unified theme (e.g., `postgres-connection-resilience`). A Package can be distributed, shared with the community, or imported into different agents to immediately inject specialized problem-solving knowledge.

### 6. Export Pack
An **Export Pack** (or **JSON Pack**) is a compiled, portable version of the agent's knowledge base. It typically takes the form of a `data.json` or `data.js` file generated by the `nve-export-dashboard` or `nve-distill` tools. It contains aggregated summaries of all Genomes, EUs, utility scores, and metrics, optimized for the Dashboard or offline viewing.

### 7. Replay Gate
A **Replay Gate** (or **Defense Gate**) is a simulated test, assertion, or validation mechanism based on historical failures. A Replay Gate acts as an automated "vaccine": it challenges the agent with a known historical failure scenario to verify if the agent's current prompt, context, or code will successfully avoid the mistake. If the agent passes, the Gate is cleared.

### 8. Utility Score
**Utility** (or **Utility Index**) is a numerical metric (e.g., 1 to 100) determining the practical value and applicability of a Genome or Skill. Utility is dynamic and increases when a pattern is successfully used (e.g., by preventing a new failure or passing a Replay Gate) and decays or remains static if unused, ensuring the agent prioritizes high-value knowledge.

## Sub-Concepts & Tooling Terms

### Auto-Distillation
The automated process by which an LLM processes raw `MEMORY.md` incidents, identifies patterns, and extracts them into structured EUs and Genomes without manual intervention.

### TAMA (Tamagotchi)
The playful codename for the overall agent evolution concept, where an agent is treated as a digital pet that "grows" and "learns" by consuming failures (Genomes) and gaining experience (XP).

### SkillGraph
The semantic network linking Genomes, EUs, and Skills together. The SkillGraph reveals relationships between seemingly disparate failures, highlighting broader structural problems or shared architectural blind spots.
