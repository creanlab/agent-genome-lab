Объяснение на пальцах

Что это такое — тремя аналогиями

📓 Тетрадь знаний

Представьте тетрадь, в которую опытный инженер записывает: «эту проблему мы уже решали, вот как». Agent Genome Lab — такая тетрадь, но машиночитаемая. AI-агент открывает её за 5 секунд и сразу знает все предыдущие уроки.

💉 Вакцина от провалов

Как вакцина учит иммунитет распознавать вирус до заражения, так Failure Genome учит агента распознавать класс ошибок до того, как они произойдут. Одна «вакцина» = одно семейство предотвращённых проблем.

📖 Кулинарная книга действий

SkillGraph — это как проверенные рецепты. Не «попробуй может поможет», а «делай так, это подтверждено на 5 инцидентах и прошло replay-тест». Навыки упакованы в пакеты и готовы к переиспользованию.

Ключевое отличие от всего остального: каждый урок здесь проверен. Replay Gate — это как клинические испытания: пока паттерн не докажет, что он работает на нескольких похожих случаях, он не попадёт в память агента. Никаких «суеверий» — только подтверждённые знания.
Проблема

Почему AI-агенты не накапливают опыт

AI-агенты по умолчанию не имеют памяти между сессиями. Каждый новый чат — чистый лист.

Агент не помнит, что вчера уже решал аналогичную задачу. Не знает, что 3 других проекта в вашей организации столкнулись с тем же паттерном. Все уроки живут в логах чата и умирают там.

Проще говоря: представьте специалиста с амнезией. Каждое утро он приходит на работу и не помнит ничего из вчерашнего дня. Он будет наступать на те же грабли снова и снова — потому что для него каждый день первый. Именно так работают все AI-агенты сегодня.

Решение

Что делает Agent Genome Lab

Genome Lab добавляет AI-агенту постоянную структурированную память. Не в базе данных, не через API — просто JSON-файлы прямо в вашем репозитории, которые любой агент может прочитать.

Agent Genome Lab — это три вещи в одном:

  • Тетрадь знаний — структурированная запись того, что произошло и что сработало
  • Лаборатория качества — каждый урок проходит replay-gate (проверку) перед тем, как стать правилом
  • Конструктор навыков — проверенные паттерны упаковываются в переиспользуемые действия
Чем это НЕ является: это НЕ база данных, НЕ промпт-библиотека, НЕ вектор-стор, НЕ баг-трекер, НЕ wiki. Это слой, где опыт становится проверяемым, переносимым и исполнимым. Файлы лежат в Git, версионируются, проходят review — как любой другой код.
Ключевой принцип: мы работаем не с отдельными проблемами, а с классами ситуаций. Один Failure Genome описывает целое семейство паттернов — и один ремонтный оператор предотвращает все вариации в будущем. Это как врач, который умеет лечить все разновидности простуды, а не каждый случай по отдельности.
Архитектура

5-уровневый конвейер опыта

Каждый рабочий эпизод проходит 5 этапов — от сырого факта до проверенного навыка, включённого в компактную память агента. Вот что происходит на каждом уровне:

1
Инцидент
Сырой факт: что произошло, почему, как решили. Записывается в JSON.
2
Experience Unit
Извлечённый урок: паттерн + анти-паттерн + верификатор.
3
Failure Genome
ДНК ситуации: семейство + ремонтный оператор + replay-результат.
4
SkillGraph
Навыки + пакеты. Admitted, quarantined, связи между навыками.
5
MEMORY.md
Компактная память: ~35 строк, агент читает за 5 секунд.
Пояснение: «Experience Unit» — это не просто записка «был баг, починили». Это извлечённый урок с чётким паттерном (как надо делать) и анти-паттерном (как НЕ надо делать). «Failure Genome» — это более зрелая версия урока, прошедшая проверку. «SkillGraph» — это уровень, где проверенные паттерны превращаются в переиспользуемые навыки, которые можно упаковать и перенести в другой проект.

Что такое Failure Genome — подробнее

Failure Genome — это как медицинский диагноз. Врач не лечит каждый случай простуды заново — он знает паттерн. Точно так же Genome Lab:

  • Контекстный отпечаток — ГДЕ это происходит (стек, файл, окружение)
  • Семейство — ЧТО за класс ситуации (не конкретная проблема, а тип)
  • Нарушенный инвариант — КАКОЕ правило было нарушено
  • Ремонтный оператор — КАК чинить этот класс ситуаций
  • Верификация — ДОКАЗАТЕЛЬСТВО что починка работает
  • Теги переносимости — ГДЕ ЕЩЁ это применимо (другие проекты)
  • Оценка полезности — НАСКОЛЬКО полезен (decay без использования, рост при применении)

Replay Gate и Admission Gate — зачем нужны два шлюза

🔄 Replay Gate

Для чего: проверяет, что геном действительно предотвращает проблемы. Это «клинические испытания» — система берёт похожие инциденты и проверяет: сработал бы этот ремонтный оператор? Если success rate ≥ 60% → promoted. Если нет → rejected.

Зачем это нужно: без проверки агент может «запомнить» ложные корреляции — суеверия. Replay Gate фильтрует мусор.

🔒 Admission Gate

Для чего: контролирует качество навыков. Навык-кандидат проходит карантин → оценку по 5 осям → и только потом получает статус «admitted». Плохие навыки отклоняются.

Зачем это нужно: память агента ограничена (~35 строк MEMORY.md). Туда должны попадать только лучшие, проверенные паттерны.

Utility Decay — почему знания «стареют»

Полезность каждого генома и навыка деградирует со временем без использования — как мышцы без тренировки:

СобытиеМножительПростое объяснение
30 дней без использования×0.9Начинает устаревать
90 дней без использования×0.7Серьёзно устарел, скоро выпадет из MEMORY
Повторное использование×1.1Подтверждает актуальность
Предотвращение проблемы×1.2Самое ценное — геном реально помогает
Негативный перенос×0.5Применили в другом контексте и навредили — штраф
Зачем это нужно: без decay MEMORY.md со временем переполнится устаревшими уроками. Utility Decay гарантирует, что агент всегда видит самые актуальные и полезные паттерны.
Применение

🌍 Не только баги — 8 сценариев использования

Agent Genome Lab работает с любым повторяемым операционным паттерном, не только с ошибками в коде. Одна и та же архитектура (incident → EU → genome → skill → package) подходит для разных доменов — меняется только контент JSON-файлов:

💻 Разработка ПО

Что сохраняется: регрессии, config drift, сломанные импорты, забытые переменные

Что переиспользуется: проверенные геномы с ремонтными операторами, паттерны исправления

🔧 DevOps / SRE

Что сохраняется: падения деплоев, rollback-сценарии, OOM-crashes, таймауты

Что переиспользуется: incident response runbooks как skill-пакеты для будущих инцидентов

🛡️ Security Operations

Что сохраняется: triage-паттерны, ложные позитивы, remediation flows

Что переиспользуется: эскалационные плейбуки, фильтры false positives

📞 Поддержка клиентов

Что сохраняется: типы запросов, причины эскалаций, решения

Что переиспользуется: reusable response skills для типичных кейсов

🔬 Исследования / Lab Ops

Что сохраняется: методология, анализ, cleaning procedures

Что переиспользуется: composable skill packages для повторных экспериментов

📋 Compliance / Аудит

Что сохраняется: evidence collection, gap remediation, acceptance criteria

Что переиспользуется: governed audit memory с evidence trails

🎓 Онбординг

Что сохраняется: типичные ошибки новичков, настройка среды, tribal knowledge

Что переиспользуется: компактный MEMORY.md — новенький агент читает его и знает все грабли

🤖 Multi-agent координация

Что сохраняется: cross-role паттерны, handoff-протоколы, конфликты

Что переиспользуется: shared skill packages между разными агентами в команде

Важно: для всех этих кейсов используется одна и та же JSON-схема и те же 18 CLI-утилит. Вам не нужно ничего менять в архитектуре — просто другой контент в тех же файлах. В комплекте уже идёт 15 готовых шаблонов-навыков для этих доменов.
Ценность

💼 Бизнес-ценность для decision-makers

Снижение затрат

73% меньше повторных проблем = меньше времени на debugging, меньше нагрузка на senior-инженеров, меньше инцидентов в production.

Ускорение онбординга

Новый агент (или новый сотрудник) читает MEMORY.md и сразу знает все предыдущие уроки. Не нужно ждать, пока он сам наступит на грабли.

Масштабирование знаний

Cross-project sharing через nve-pack: знания из проекта A безопасно переносятся в проект B. Organizational intelligence растёт с каждым проектом.

Аудируемость

Все решения прослеживаемы: incident → lesson → genome → skill. Каждый шаг записан в JSON с timestamp. CI/CD gate через nve-audit --ci.

Приватность и контроль

Local-first: данные не покидают ваш репо. 4-уровневая редакция при экспорте. Никаких облачных зависимостей — всё под вашим контролем.

Интеграция

🖥️ VS Code Extension — визуальное управление

Рекомендуемый способ работы. Расширение добавляет боковую панель с 6 интерактивными разделами. Всё обновляется автоматически при изменении JSON.

📊 Audit Dashboard

5-осевой аудит + SkillGraph score. Прогресс по каждой оси в реальном времени.

🧬 Genome Families

Дерево Failure Genomes по семействам. Клик → открывает JSON в редакторе.

🔄 Replay Gate

promoted ✅ / rejected ❌ / pending ⏳. Pass rate и размер семейства в tooltip.

🧩 Skill Registry

Навыки по статусу: admitted, candidate, quarantined, rejected. С оценками.

📦 Skill Packages

Пакеты с раскрываемым списком навыков. Категория и путь публикации.
